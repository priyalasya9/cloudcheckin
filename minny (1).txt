PART 1 — Create Linux EC2 & Connect Using PuTTY (.ppk)
🔹 A. Create Ubuntu EC2
Go to AWS Console
Go to EC2 → Launch Instance
Name: Linux-EC2
AMI: Choose Ubuntu
Instance type: t2.micro
Key pair:
Create new key pair
Download .pem file
Network settings:
Allow SSH (Port 22)
Source: My IP
Click Launch
Wait until instance state = Running.

🔹 B. VERY IMPORTANT — Check Security Group
Go to:
EC2 → Security Groups → Inbound Rules
Make sure:
Type: SSH
Port: 22
Source: My IP
If not → Add rule.

🔹 C. Convert .pem to .ppk (Only for Windows)
PuTTY cannot use .pem directly.
Open PuTTYgen
Click Load
Select your .pem
Click Save Private Key
Save as .ppk
Done.

🔹 D. Connect Using PuTTY
Open PuTTY
Host Name:
ubuntu@<Public-IP>
Example:
ubuntu@3.xx.xx.xx
Go to:
Connection → SSH → Auth → Credentials
Click Browse
Select your .ppk
Click Open
Click Yes (first time warning)
You are connected 🎉

🔹 VERIFY
Type:
whoami
Output:
ubuntu
Done Linux via PuTTY ✅

🌿 WEEK 3 – Amazon EFS (Shared Storage)
🧠 What is EFS?

EFS = Elastic File System

👉 Shared storage
👉 Multiple EC2 instances can access same files
👉 Uses NFS (port 2049)

🔥 PART 1 — Create 2 EC2 Instances
✅ Step 1: Launch First EC2

Name: EFS-1

AMI: Amazon Linux (important)

Instance type: t2.micro

Key pair: select existing

Security group:

SSH (22) → My IP

Add rule:

Type: NFS

Port: 2049

Source: 0.0.0.0/0 (lab purpose)

Launch.

✅ Step 2: Launch Second EC2

Same configuration.

Name: EFS-2

Make sure both are:

✔ Running
✔ 2/2 checks passed

🔥 PART 2 — Create EFS File System
✅ Step 3: Create EFS

Go to EFS service

Click Create file system

Choose default VPC

Click Next until Review

Create

Important:
Make sure security group attached allows NFS (2049).

🔥 PART 3 — Connect to EC2 & Mount EFS

Now open 2 terminals (PuTTY or CloudShell).

✅ Step 4: SSH into EFS-1
ssh -i key.pem ec2-user@public-ip

Become root:

sudo su
✅ Step 5: Install EFS Utils
yum install -y amazon-efs-utils
✅ Step 6: Create Mount Directory
mkdir /efs
✅ Step 7: Attach EFS

Go to:

EFS → Click your file system → Click Attach

Copy mount command (DNS method)

It will look like:

mount -t efs fs-xxxx:/ /efs

Paste in terminal.

🔥 PART 4 — Test Shared Storage

On EFS-1:

cd /efs
touch testfile.txt

Now go to EFS-2:

Repeat:

sudo su
yum install -y amazon-efs-utils
mkdir /efs
mount -t efs fs-xxxx:/ /efs
cd /efs
ls

You should see:

testfile.txt

🎉 Shared storage working.
💾 WEEK 2 – EBS (Ultra Clear Version)

This week has 3 parts:

1️⃣ Create & Attach Volume
2️⃣ Permanently Mount (fstab)
3️⃣ Snapshot → Copy to another region

🔥 PART 1 – Create & Attach EBS
✅ Step 1 – Create Volume

EC2 → Volumes → Create Volume

Type → gp3

Size → 20GB

AZ → SAME as EC2

Click Create.

⚠️ Volume & EC2 must be in same AZ.

✅ Step 2 – Attach Volume

Select volume → Actions → Attach

Select EC2 instance

Device name → /dev/xvdf

Attach.

✅ Step 3 – Verify in EC2

SSH into instance:

lsblk

You will see something like:

/dev/nvme0n1 → root

/dev/nvme1n1 → new disk

Remember:
Console shows /dev/xvdf
But Linux shows nvme1n1.

🔥 PART 2 – Permanent Mount (Important)

This is the main exam part.

✅ Step 4 – Partition
sudo fdisk /dev/nvme1n1

Inside fdisk:

n
p
Enter
Enter
Enter
w

Then:

sudo partprobe
✅ Step 5 – Format
sudo mkfs.xfs /dev/nvme1n1p1
✅ Step 6 – Create Mount Folder
sudo mkdir /mnt/data
✅ Step 7 – Mount
sudo mount /dev/nvme1n1p1 /mnt/data

Check:

df -h
✅ Step 8 – Make Permanent (VERY IMPORTANT)

Get UUID:

sudo blkid /dev/nvme1n1p1

Copy UUID.

Edit fstab:

sudo nano /etc/fstab

Add line:

UUID=your-uuid /mnt/data xfs defaults,nofail 0 0

Save.

Test:

sudo mount -a
✅ Step 9 – Reboot Test
sudo reboot

After reconnect:

df -h

If mounted automatically → Success.

✔ Database has NO public IP

⚠️ Common Errors + Fix

❌ Permission denied
→ Run:

chmod 400 dbserver.pem

❌ Connection timed out
→ Check DB security group allows SSH from Bastion SG

❌ Bastion not connecting
→ Check:

Public IP exists

Route table has IGW

SSH allowed in SG
