# WEEK 6 FULL REVISION — VPC WITH BASTION SERVER + PRIVATE SERVER + NAT GATEWAY

Based on your lab document: 

---

# AIM

To create a secure AWS architecture where:

* Bastion Server is in Public Subnet
* Private EC2/Database Server is in Private Subnet
* Private server cannot be accessed directly from internet
* NAT Gateway provides internet access to private subnet

---

# FINAL ARCHITECTURE

```text id="jlwmx1"
Internet
   ↓
Internet Gateway
   ↓
Public Route Table
   ↓
Public Subnet (10.0.1.0/24)
   ↓
Bastion Server
   ↓
Private Subnet (10.0.2.0/24)
   ↓
Private EC2 / DB Server
   ↓
NAT Gateway → Internet
```

# VERY IMPORTANT VIVA QUESTIONS

## 1. What is Bastion Server?

A Bastion Server is a public EC2 instance used to securely access private EC2 instances.

---

## 2. Why is private EC2 secure?

Because it has no public IP and cannot be directly accessed from internet.

---

## 3. Why is NAT Gateway needed?

To provide outbound internet access for private subnet instances.

---

## 4. Difference between Bastion and NAT Gateway?

| Bastion      | NAT Gateway              |
| ------------ | ------------------------ |
| Used for SSH | Used for internet access |
| Public EC2   | Managed AWS service      |
| Admin access | Outbound traffic         |

---

## 5. Why must NAT Gateway be in public subnet?

Because it requires Internet Gateway connectivity.

---

## 6. Why does SSH timeout happen?

Possible reasons:

* Missing SG rule
* Missing IGW route
* No public IP
* Wrong subnet association

---

## 7. Why should private SG not allow 0.0.0.0/0?

Because it exposes private EC2 to the internet.

---

## 8. Can Bastion replace NAT Gateway?

❌ No.

Bastion provides SSH access only.
NAT Gateway provides internet access.
